Naming Conventions

Files:
File names must only include losercase letters, numbers, and dashes. Dashes are word separators, while underscores are word connectors.

Good Examples:

  • chocolate-cake-recipe.html
  • west-coast-art-2021.jpeg
  • snow_white-apple.jpeg

Bad Examples:

  • pureKitchenLogo.svg
  • Today-in-History.html
  • picture of black cat.jpg
Paths:
The purpose of a file path is to include a file within your website. Paths open and close on their own.

<img src="image link" alt="image description>"

Adds an image to website. If image doesn't load, image description will act as a placeholder. In addition, the image description is read by text to speech for the visually impaired.

Commit Messages

Best practices

  • Minimum 3 words and 10 characters
  • Proper spelling and grammar
  • Capital start letter
  • No period
  • Must start with imperative verb (ex. Create, Fix, Add, etc.)
  • Describe as if starting with "This commit will..."

Common HTML Tags

Headings

Heading tags mark headings based on importance.

<h#> insert heading text here</h#>

This is an <h1> heading. It is the most important, so there should only be one per page.

This is an <h2> heading.

This is an <h3> heading.

This is an <h4> heading.

This is an <h5> heading.
This is an <h6> heading. It is the least important.

Paragraphs

Defines a paragraph of text. Always starts on a new line of code. Browser automatically adds margins above and below defined text.

<p>insert text here</p>

List Item

Individual items within a list.

<li>Item name</li>

Unordered list

List without a numbered order. Opens and closes on its own line. Contains list items that begin with a dot.

<ul>

<li>item name here</li>

</ul>

Ordered list

List in a numbered order. Opens and closes on its own line. Contains list items that begin with a designated number.

<ol>

<li>item name here</li>

</ol>
